Justificación del Formulario

Nombre Usuario

Se requiere para identificar al usuario. Debe tener entre 3 y 16 caracteres y solo puede contener letras, números y guiones bajos. Decidimos usar un campo de texto con una expresión regular para asegurar que el nombre de usuario cumpla con los requisitos.

Contraseña

Se requiere para la seguridad del usuario. Debe tener al menos 6 caracteres. Usamos un campo de contraseña con una expresión regular para asegurar la longitud mínima y mejorar la seguridad.

Repetir Contraseña

Se requiere para verificar que el usuario haya ingresado correctamente su contraseña. Este campo ayuda a prevenir errores en la entrada de la contraseña.

Dirección

Se utiliza para la entrega de servicios o productos. Un campo de texto simple permite la entrada libre para la dirección del usuario.

Ciudad

Se utiliza para identificar la ubicación del usuario. Debe contener al menos 2 caracteres y solo letras. Usamos una expresión regular para asegurar que solo se ingresen letras.

Provincia

Se utiliza para identificar la región del usuario. Debe contener al menos 2 caracteres y solo letras. Similar al campo de ciudad, usamos una expresión regular para asegurar la entrada correcta.

Número de Huéspedes

Se requiere para saber la cantidad de personas que se hospedarán. Usamos un campo numérico para limitar la entrada a valores válidos.

Servicios brindados

Opción para seleccionar los diferentes servicios que nosotros le brindamos. Usamos checkboxes para permitir múltiples selecciones.

Tipo de habitación

Opción para seleccionar las diferentes habitaciones disponibles. Utilizamos botones de radio para limitar la selección a una opción.

Tipo de pensión

Opción para seleccionar el tipo de pensión. Similar a las habitaciones, usamos botones de radio para una selección exclusiva.

Comentarios

Opción para enviar su correo y dejar algún comentario o sugerencia. Usamos un textarea para permitir una entrada de texto más larga.
